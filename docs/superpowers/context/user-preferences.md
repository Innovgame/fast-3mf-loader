# 用户偏好

这是仓库级用户工作习惯摘要。新会话在开始任务前，应与仓库背景信息一起阅读本文件。

## 默认工作方式

- 默认在任务完成后帮用户创建 git commit，但前提是这次改动具备清晰、可独立提交的边界。
- 如果任务只是讨论、分析、答疑或尚未形成稳定改动，不要为了提交而制造无意义 commit。
- 除非用户明确要求，否则不要开启 sub-agent。
- 默认优先在当前会话内直接完成任务，不把实现拆给其他 agent。
- 默认在当前工作目录内继续执行任务，不主动创建 git worktree 或切换到额外隔离工作区。
- 只有用户明确要求，或当前工作目录状态已经明显妨碍任务推进时，再考虑使用 worktree 等隔离方式。

## 文档语言偏好

- 默认使用中文编写新增文档、计划、说明和执行反馈。
- 如果需要英文版本，后续补同名 `.en.md` 文档，中文为主文档。
- 不要在同一份文档中中英混写，除非是代码标识符、命令、路径、类型名或必要引用。

## 何时必须更新本文件

- 用户明确提出新的长期工作习惯
- 现有偏好与仓库中的自动化流程不再一致
- 新的工作规则取代了旧规则
